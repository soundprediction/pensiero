Fix: document packs as metadata-only until backend semantics and tests exist, or implement the primitive subset first.

- **Critical: the revised phase order only partially landed.** The correction block puts safety before autonomy [docs/SELF_OPTIMIZING_DAEMON.md](/home/josh/workspace/pensiero/docs/SELF_OPTIMIZING_DAEMON.md:47), but the main phased plan still schedules idle optimization before validation/hot-swap [docs/SELF_OPTIMIZING_DAEMON.md](/home/josh/workspace/pensiero/docs/SELF_OPTIMIZING_DAEMON.md:578), keeps the bandit [docs/SELF_OPTIMIZING_DAEMON.md](/home/josh/workspace/pensiero/docs/SELF_OPTIMIZING_DAEMON.md:584), and leaves handle pools in Phase 7 [docs/SELF_OPTIMIZING_DAEMON.md](/home/josh/workspace/pensiero/docs/SELF_OPTIMIZING_DAEMON.md:592) despite the single per-handle mutex [ladybug_system.go](/home/josh/workspace/pensiero/cmd/pensiero/ladybug_system.go:50). Fix: rewrite the main phased plan, not just the correction preamble.

- **Critical: domain/range constraints are not currently implementable or sound as hard logic.** `PredicateMeta` has no domain/range [predicates.go](/home/josh/workspace/pensiero/pkg/reasoning/predicates.go:25), the CLI loader does not parse them [registry.go](/home/josh/workspace/pensiero/cmd/pensiero/registry.go:19), and `GraphQuerier` is only raw `Query` [types.go](/home/josh/workspace/pensiero/pkg/reasoning/types.go:103). Emitted labels are `scope/concept/endpoint`, not semantic entity types [emitter.go](/home/josh/workspace/pensiero/pkg/generalization/emitter.go:110). Also, the doc says a lifted parent must satisfy `range`, but the parent is the relation head, so that is a `domain` check [docs/SELF_OPTIMIZING_DAEMON.md](/home/josh/workspace/pensiero/docs/SELF_OPTIMIZING_DAEMON.md:147). Fix: start with explicit type packs and soft validation; use domain for subject/head and range for object/tail; do not fire contradictions until there is closed-world type provenance.

- **Critical: provisional/speculative safety still has no storage enforcement.** The doc says speculative output and approximate types cannot corrupt live logic [docs/SELF_OPTIMIZING_DAEMON.md](/home/josh/workspace/pensiero/docs/SELF_OPTIMIZING_DAEMON.md:238), [docs/SELF_OPTIMIZING_DAEMON.md](/home/josh/workspace/pensiero/docs/SELF_OPTIMIZING_DAEMON.md:481), but current serving and IGL queries do not filter status/provenance, and `ExcludeDeduced` is unused [types.go](/home/josh/workspace/pensiero/pkg/reasoning/types.go:77), [queries.go](/home/josh/workspace/pensiero/pkg/generalization/queries.go:224). Fix: add storage-level provenance/status and default exclusion before any cognition, type inference, or overlay promotion.

- **High: the predicate-pack loader is much less ready than “80% shaped.”** There is no `RegisterPack`, `BuildRegistry`, `extends`, registry fingerprint, or metadata enumeration. `loadRegistry` is `general` or file only [registry.go](/home/josh/workspace/pensiero/cmd/pensiero/registry.go:28). Alias resolution is file-local, override semantics can leave stale raw aliases, and proposed hard validation would fail current medical disjoints because `contraindicated`, `indicated`, and `prevents` are not declared canonical predicates [predicate_primitives.go](/home/josh/workspace/pensiero/pkg/reasoning/predicate_primitives.go:85). Fix: build pack validation first, disallow built-in overrides by default, expose `Fingerprint()`/`All()`, and repair the medical pack.

- **High: predicate inventory is a large new subsystem, not a small adapter method.** The doc requires distinct predicates plus head/tail type distributions [docs/SELF_OPTIMIZING_DAEMON.md](/home/josh/workspace/pensiero/docs/SELF_OPTIMIZING_DAEMON.md:197), but neither `GraphQuerier` nor the proto exposes this [reasoning.proto](/home/josh/workspace/pensiero/proto/pensiero/v1/reasoning.proto:10). On a 227GB-class graph, startup full scans will be expensive and stale immediately after ingest. Fix: make inventory an offline/incremental materialized stats job with sampling/watermarks first; expose it later through a v2 API.

Centroid-nearest-supertype is a weak semantic guess and fails on ambiguous labels, mixed clusters, model drift, and missing labels. Fix: explicit type packs first; embeddings may generate suggestions/questions only, keyed by model/version/confidence, and must not affect cache keys or live ranking unless confirmed.

**Recommended Ordering**
Minimal viable sequence: fix predicate-correct entailment, then static gRPC daemon with immutable generation store and read-only handle pool, then registry packs/fingerprint, then provenance/quarantine plus validated symlink hot-swap, then telemetry/cache, then idle IGL. After that, add explicit-only type packs and soft domain/range checks. Inventory should be incremental/offline before it is live. Embedding type models and question generation belong last.

**Verdict**
The revised plan is directionally better, but still not sound as ordered. The new knowledge-model layer is ahead of the safety and semantics foundations: packs are useful early only as validated registry data, while domain/range, inventory, and embedding type models should remain non-blocking advisory features until predicate-correct reasoning, provenance quarantine, and validated hot-swap are real.
