# codex design review — SELF_OPTIMIZING_DAEMON.md (2026-06-23, read-only against the code)

No files edited.

**Findings**

- **Critical: Predicato integration claim is false.** The doc says `pkg/connector/predicato.go` supplies `GraphQuerier`, embeddings, and hybrid search, but the actual `PredicatoClient` only calls `/api/v1/extract` and writes extracted triples/rules into a local repo: [predicato.go](/home/josh/workspace/pensiero/pkg/connector/predicato.go:72). It has no `Query`, `Embed`, or search method. Fix: split this into explicit adapters: `PredicatoGraphQuerier`, `PredicatoEmbedder`, and `PredicatoSearchClient`, or remove those claims from Phase 6.

- **Critical: `cmd/pensiero serve` does not serve gRPC.** The serve path only opens a source graph, builds a `generalization.Loop`, starts HTTP health/metrics, and runs IGL: [serve.go](/home/josh/workspace/pensiero/cmd/pensiero/serve.go:53), [serve.go](/home/josh/workspace/pensiero/cmd/pensiero/serve.go:95), [serve.go](/home/josh/workspace/pensiero/cmd/pensiero/serve.go:121). Fix: Phase 1 must add `--grpc-addr`, snapshot graph loading, reasoner construction, `grpcsvc.Serve`/registration, readiness, and graceful shutdown.

- **Critical: Engine hot-swap premise is wrong as written.** `Engine` holds one private `GraphQuerier` field set by `NewEngine`: [engine.go](/home/josh/workspace/pensiero/pkg/reasoning/engine.go:15), [engine.go](/home/josh/workspace/pensiero/pkg/reasoning/engine.go:22). There is no atomic pointer or setter. You can only hot-swap by swapping the whole `Reasoner` generation, or by passing an atomic wrapper that itself implements `GraphQuerier`. Fix: design `GenerationStore` around swapping whole immutable `Reasoner` + graph handles, not mutating `Engine`.

- **Critical: built-in `Engine.Entails` ignores `Claim.Predicate`.** It calls `Derive` with only source/target and `IncludeInverse`, so any path between subject/object can entail any predicate: [engine.go](/home/josh/workspace/pensiero/pkg/reasoning/engine.go:65). The design’s cache keys, questions, and validation all assume claim-shaped predicate semantics. Fix: either fix `Engine.Entails` to constrain/validate the predicate, or explicitly serve `NativeReasoner` and document backend-specific semantics.

- **Critical: speculative overlay safety is not backed by code.** The doc promises quarantine before live answers: [SELF_OPTIMIZING_DAEMON.md](/home/josh/workspace/pensiero/docs/SELF_OPTIMIZING_DAEMON.md:238), but current graph queries do not filter source status, deduced facts, or speculative namespaces; `ExcludeDeduced` exists but is unused: [types.go](/home/josh/workspace/pensiero/pkg/reasoning/types.go:77), [queries.go](/home/josh/workspace/pensiero/pkg/generalization/queries.go:140). Fix: define storage-level status/provenance, make reasoning and IGL exclude speculative/deduced edges by default, and only promote via an explicit transaction/manifest.

- **High: `reasoning.Reasoner` is the right drop-in interface, but too narrow for daemon lifecycle.** `grpcsvc.Server` can wrap any `Reasoner`: [server.go](/home/josh/workspace/pensiero/pkg/grpcsvc/server.go:20). However, the proto has no scope, generation, budget, or question streaming fields: [reasoning.proto](/home/josh/workspace/pensiero/proto/pensiero/v1/reasoning.proto:47). Fix: keep `Reasoner` for query compatibility, but put scope/budget in metadata or add a v2 API before multi-scope routing and active learning.

- **High: publish is not simply temp-to-final rename.** Files use `os.Rename`, but directory snapshots are staged to a `.snap` dir and final path becomes an atomically swapped symlink: [igl.go](/home/josh/workspace/pensiero/pkg/generalization/igl.go:305). Fix: document symlink/version semantics and design reload/watch/cleanup around versioned directories.

- **High: cancellation can still publish.** `Publisher.Publish` writes, validates, then calls `publishSnapshot` without checking `ctx.Err()` immediately before publish: [igl.go](/home/josh/workspace/pensiero/pkg/generalization/igl.go:121). If load arrives after validation, the canceled pass can still swap output. Fix: check context before publish and make the scheduler’s cancellation contract explicit.

- **High: builder yield/cancellation is overstated.** IGL has query calls where a wrapper can yield, but CPU loops in taxonomy assembly and graph assembly do not check context: [builder.go](/home/josh/workspace/pensiero/pkg/generalization/builder.go:25), [queries.go](/home/josh/workspace/pensiero/pkg/generalization/queries.go:45). Fix: add cooperative context checks in long loops before relying on idle cancellation.

- **High: live serving would serialize on a single graph handle.** `ladybugGraph.Query` locks a per-handle mutex for every query: [ladybug_system.go](/home/josh/workspace/pensiero/cmd/pensiero/ladybug_system.go:50). Phase 7 puts handle pools too late. Fix: Phase 1 should include a small read-only handle pool per generation, or gRPC concurrency will bottleneck immediately.

- **High: backend contradiction semantics diverge and leak domain assumptions.** `Engine.Contradicts` uses ontology class disjointness: [engine.go](/home/josh/workspace/pensiero/pkg/reasoning/engine.go:94). `NativeReasoner.Contradicts` uses disjoint predicates and medical/drug base-name matching: [native.go](/home/josh/workspace/pensiero/pkg/reasoning/native.go:103). Fix: choose one serving backend per generation and validate against that backend; move medical heuristics out of generic pensiero.

- **High: dependency boundary is leaky in the doc.** The module does not import humn, but the design bakes humn/DDx/800ms/condition-finding assumptions into core phases and metrics: [SELF_OPTIMIZING_DAEMON.md](/home/josh/workspace/pensiero/docs/SELF_OPTIMIZING_DAEMON.md:27), [SELF_OPTIMIZING_DAEMON.md](/home/josh/workspace/pensiero/docs/SELF_OPTIMIZING_DAEMON.md:141). Fix: move humn measurements to an appendix; make SLO, golden sets, and query labels caller-supplied generics.

- **High: actual pensiero is not fully domain-agnostic today.** Medical predicates are in core reasoning: [predicate_primitives.go](/home/josh/workspace/pensiero/pkg/reasoning/predicate_primitives.go:3), and Cypher filters “clinical subgraph” labels: [cypher.go](/home/josh/workspace/pensiero/pkg/reasoning/cypher.go:31). Fix: make the daemon plan include domain decontamination or stop claiming strict domain agnosticism.

- **Medium: cache invalidation plan needs more inputs.** Generation ID alone is insufficient; results depend on backend, registry, scope route, graph path, config, request params, and whether timeouts/errors are cached. There is also no registry fingerprint API; registry internals are unexported: [predicates.go](/home/josh/workspace/pensiero/pkg/reasoning/predicates.go:49). Fix: add an explicit stable `Registry.Fingerprint()` and never cache errors/timeouts as logical results.

- **Medium: idle detection can race and livelock.** The doc’s `in_flight == 0 for quiet_for` can start a pass just as a query begins, and cancel-on-load can repeatedly throw away full passes: [SELF_OPTIMIZING_DAEMON.md](/home/josh/workspace/pensiero/docs/SELF_OPTIMIZING_DAEMON.md:98). Fix: use an atomic load epoch/lease, chunked work, cooldowns, and publish rate limits.

- **Medium: bandit reward is not well-defined.** “validated derivation/new lift/later cache hit” mixes delayed, non-stationary, and differently priced outcomes: [SELF_OPTIMIZING_DAEMON.md](/home/josh/workspace/pensiero/docs/SELF_OPTIMIZING_DAEMON.md:169). Fix: start with fixed heuristics; if a bandit is added, use cost-normalized rewards after validation only, with decay and attribution windows.

- **Medium: embeddings/hybrid search are both absent and under-costed.** The doc assumes predicato `/v1/embeddings` and hybrid search: [SELF_OPTIMIZING_DAEMON.md](/home/josh/workspace/pensiero/docs/SELF_OPTIMIZING_DAEMON.md:204), but no such pensiero client exists. Fix: add batching, singleflight, persistent cache keyed by model+text, rate limits, and ANN rebuild/invalidation before semantic exploration.

- **Medium: serving-loop snapshots may use the wrong embedding schema.** `build-generalization` detects source embedding dimension, but `serve`’s `ladybugSnapshotWriter` always uses emitter default 1024: [serve.go](/home/josh/workspace/pensiero/cmd/pensiero/serve.go:150), while dimension detection exists elsewhere: [main.go](/home/josh/workspace/pensiero/cmd/pensiero/main.go:181). Fix: pass source embedding dimension into the serve writer, or remove embedding-index claims from loop output.

**Verdict**

The phased direction is not sound as ordered. Phase 1 should be a static gRPC daemon with one loaded generation, readiness, and a handle pool. Phase 2 can add telemetry/cache. Then add manual validation + generation hot-swap before any idle optimizer. Only after safe swaps exist should `Loop.RunOnce` be scheduled when idle. Query-driven IGL comes after that. Predicato embedding/search adapters and the whole cognition/question layer should be last; right now they are under-specified and unsafe relative to the actual code.
